# BrainySoft External Services — Bruno Workspace

Данный репозиторий представляет собой **рабочее пространство (Workspace)** для API-клиента [Bruno](https://www.usebruno.com/). Здесь собраны коллекции запросов к внешним сервисам, которые мы используем в BrainySoft.

## 🚀 Как начать работу (для новых пользователей)

Чтобы начать работу с этим репозиторием, вам не нужно создавать коллекции вручную — достаточно открыть его как готовое рабочее пространство.

### 1. Установка Bruno
Если у вас еще нет Bruno, скачайте его с [официального сайта](https://www.usebruno.com/downloads). Это бесплатный Open-source инструмент, который хранит данные локально в вашей файловой системе.

### 2. Открытие рабочего пространства
1. Запустите приложение Bruno.
2. В левом верхнем углу нажмите на название текущего воркспейса (или на иконку папки/меню).
3. Выберите пункт **"Open Workspace"**.
4. В появившемся окне выбора папок выберите **корневую директорию этого репозитория** (ту, где лежит файл `workspace.yml`).
5. Bruno автоматически подтянет все вложенные коллекции (например, Infotex).

### 3. Настройка переменных и секретов
Для безопасности мы не храним пароли и ключи в репозитории.
1. В интерфейсе Bruno выберите нужную коллекцию (например, **Infotex**).
2. Нажмите на выпадающий список окружений (справа сверху, обычно там написано "No Environment").
3. Выберите нужное окружение (например, **Prod**).
4. Если запросы требуют авторизации, создайте в папке коллекции файл `.env` (на базе приложенного `.env.example`) и впишите туда свои доступы. Bruno автоматически подставит их в переменные вида `{{process.env.VAR_NAME}}`.

## 📂 Структура воркспейса
Рабочее пространство организовано так, чтобы каждая интеграция была изолирована:
- **Collections/** — здесь лежат папки с конкретными API (Infotex и др.). Каждая папка — это отдельная коллекция со своими запросами и настройками.
- **Environments/** — внутри каждой коллекции есть свои настройки окружений (адреса серверов, префиксы).

## 🛠 Основные принципы

1. **Текстовый формат:** Все запросы хранятся в формате `.bru` или `.yml`. Это позволяет нам использовать Git для совместной работы — вы можете видеть историю изменений каждого запроса.
2. **Безопасность прежде всего:** Никогда не вписывайте реальные пароли или токены прямо в поля запросов. Используйте переменные окружения и файл `.env`.
3. **Документация "из коробки":** У каждого запроса в Bruno есть вкладка **Docs**. Мы стараемся описывать там назначение полей и особенности работы эндпоинта.
4. **Синхронизация через Git:** Чтобы получить новые запросы от коллег, просто сделайте `git pull`. Bruno мгновенно обновит список запросов в интерфейсе.
